Ceci est un template de projet Angular (ICDC)

Stack technique

  • Angular v7+
  • Webpack v4+
  • Typescript 3.1.3
  • Ngx-translate 11.0.1 - doc
  • Angular CLI : 7.1.0
  • Tests unitaires : Mocha, Chai, Sinon
  • Tests IHM : Protractor !!

Documentation

  • Environnement de développement
  • Détails des fichiers générés
  • Tests unitaires avec Mocha, Chai et Sinon

Le sous-générateur yo icdc:pwa est disponible pour convertir une application Angular en une application PWA. On se base sur les plugins suivants :

  • Workbox webpack Plugins
  • Webpack pwa manifest Plugin

La configuration de ces plugins est externalisée dans les fichiers : manifest-config.js et sw-config.js.

Déployer le projet sur le serveur de développement local, l'apllication est disponible sur localhost:3000 :

npm start

Construire un artéfact optimisé pour la production et prêt à être déployé sur un serveur de :

npm run build:package

Lancer un serveur express avec le bundle construit sur le repretoire dist :

npm run server:prod

Lancer les tests unitaires avec Mocha :

npm run test

Lancer les tests IHM protractor :

npm run e2e

Lancer les tests unitaires en mode serveur sur le port 8888 (http://localhost:8888, utile pour débugger ses TU) :

npm run test:server

Lancer l'analyse tslint + codelyzer. Les régles sont dans le fichier tslint.json :

npm run lint

Le projet est compatible Angular CLI (sauf pour la build et le lancement - ng serve et ng build)

Génère un composant Angular dans le répertoire courant. Changer component par (directive, pipe, service, class, guard, interface, enum, module) :

ng g component mon-super-composant
